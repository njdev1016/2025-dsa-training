# DSA Training Plan for 2025 HS October Challenge

## Problem Analysis by Topics:

**Part 01** - Frequency counting with sorting
**Part 02** - Basic arithmetic & modulo
**Part 03** - String manipulation & unique characters
**Part 04** - 2D grid navigation
**Part 05** - Sliding window algorithm
**Part 06** - Mathematical operations (Euclidean distance)
**Part 07** - Number systems & binary conversion
**Part 08** - Anagram detection with hash maps
**Part 09** - BFS shortest path (most complex)
**Part 10** - Counting & pairing elements

## Recommended Learning Path (6-8 Weeks)

### Week 1: Fundamentals (12-15 hours total, 1.5-2 hours/day)

**Days 1-3: Array & String Basics (4-6 hours)**
- Fixed vs Dynamic arrays (1 hour)
- Array traversal & iteration (1 hour)
- String basics & immutability (1 hour)
- Practice: 5-8 easy problems on arrays/strings

**Days 4-5: Hash Maps/Dictionaries (3-4 hours)**
- Hash map concepts & operations (1.5 hours)
- Frequency counting patterns (1.5 hours)
- Practice: 5-6 frequency counting problems

**Days 6-7: Hash Sets (3-4 hours)**
- Set operations & uniqueness (1.5 hours)
- Character/element deduplication (1.5 hours)
- Practice: 4-5 set-based problems

**Ready to solve:** Part 03, Part 10

---

### Week 2: Sorting & Advanced Hashing (12-15 hours, 1.5-2 hours/day)

**Days 8-10: Sorting Algorithms (4-5 hours)**
- Comparison-based sorting (2 hours)
- Sorting with custom comparators (1.5 hours)
- Practice: 5-7 sorting problems

**Days 11-12: Advanced Hash Map Patterns (3-4 hours)**
- Sorting characters for anagrams (1.5 hours)
- Hash map with complex keys (1.5 hours)
- Practice: 4-5 anagram/grouping problems

**Days 13-14: Review & Integration (4-5 hours)**
- Combined hash map + sorting problems (2 hours)
- Practice: 6-8 medium problems

**Ready to solve:** Part 01, Part 08

---

### Week 3: Mathematical Operations (10-12 hours, 1.5-2 hours/day)

**Days 15-16: Modulo Arithmetic (3-4 hours)**
- Modulo properties & operations (1.5 hours)

- Practice: 4-5 modulo problems

**Days 17-18: Number Systems (3-4 hours)**
- Binary, decimal, hexadecimal (1.5 hours)
- Conversion algorithms (1.5 hours)
- Practice: 5-6 conversion problems

**Days 19-20: Mathematical Formulas (3-4 hours)**
- Distance formulas (Euclidean, Manhattan) (1.5 hours)

- Practice: 4-5 geometry/math problems

### Week 4: 2D Grids & Matrix Operations (12-15 hours, 1.5-2 hours/day)

**Days 24-26: Grid Navigation (6-8 hours)**

---

### Week 5: Sliding Window (10-12 hours, 1.5-2 hours/day)

- Practice: 4-5 fixed window problems

**Days 29-31: Variable Window (6-8 hours)**

### Week 6: Graph Theory Basics (15-18 hours, 2-2.5 hours/day)

**Days 35-38: BFS Algorithm (8-10 hours)**

### Week 7-8: Integration & Challenge Solving (20-25 hours, 1.5-2 hours/day)

**Days 39-52: Solve All Challenges**

## Recommended Challenge Solving Order:

1. **Part 03** (Game Begins) - Easiest, just find unique letters
2. **Part 10** (Good Ending) - Character counting & pairing
3. **Part 02** (White Noise) - Simple modulo arithmetic
4. **Part 01** (Wrong Return) - Frequency counting + sorting
5. **Part 06** (Runtime Warp) - Math formulas
6. **Part 07** (Phone Call) - Binary conversion
7. **Part 08** (Eldritch Horror) - Anagram detection
8. **Part 04** (Kitchen Secret) - Grid navigation
9. **Part 05** (Piano Puzzle) - Sliding window (challenging)
10. **Part 09** (You Cannot Escape) - BFS shortest path (hardest)

---

## Time Commitment Summary:

- **Weeks 1-6:** Learning fundamentals (71-87 hours)
  - Average: 1.5-2 hours/day
- **Weeks 7-8:** Solving challenges (20-25 hours)
  - Average: 1.5-2 hours/day

**Total:** 8 weeks, ~100-110 hours (1.5-2 hours/day average)

---

## Key Success Tips:

1. Don't skip prerequisites - each week builds on the previous
2. Practice problems are crucial - aim for 80% accuracy before moving on
3. Use platforms like LeetCode, HackerRank for practice problems
4. Read sample inputs to understand constraints
5. Test edge cases thoroughly
6. Week 7-8 buffer allows time for debugging and learning from mistakes

## Challenge Details:

### Part 01 - Wrong Return
Find the 4 most common digits in a sequence. If tied, higher digit comes first.

**Topics:** Frequency counting, Hash maps, Sorting

---

### Part 02 - White Noise
Calculate total rotation in degrees (0-359) from a sequence of rotations.

**Topics:** Modulo arithmetic, Array traversal

---

### Part 03 - Game Begins
Find unique letters in text.

**Topics:** String manipulation, Hash sets

---

### Part 04 - Kitchen Secret
Navigate a 4x4 keypad grid following UP/DOWN/LEFT/RIGHT directions.

**Topics:** 2D grid navigation, Boundary checking

---

### Part 05 - Piano Puzzle
Find shortest substring containing all 7 notes (A-G).

**Topics:** Sliding window, Hash map

---

### Part 06 - Runtime Warp
Calculate total Euclidean distance between 4D vectors, rounding up non-integers.

**Topics:** Mathematical operations, Distance formulas

---

### Part 07 - Phone Call
Convert decimal to binary with 32-bit overflow handling.

**Topics:** Number systems, Binary conversion, Bit manipulation

---

### Part 08 - Eldritch Horror
Find the word that has no anagram in the dataset.

**Topics:** Anagram detection, Hash maps, String sorting

---

### Part 09 - You Cannot Escape
Find shortest path in a grid from P to G avoiding walls (#).

**Topics:** BFS, Graph traversal, Shortest path

---

### Part 10 - Good Ending
Find 2 unpaired letters (one from left 8, one from right 8) in a 16-letter string.

**Topics:** Character counting, Hash maps
